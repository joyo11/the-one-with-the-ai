- **Phase 2 training complete.** DistilBERT fine-tuned 3 epochs on M2 MPS, 7203s (2 hr). Val accuracy by epoch: 30.1% → 32.0% → 32.3% (loss plateau at 1.65 vs random 1.79). Best model saved to `classifier/models/best/`.
- **Phase 2 eval complete.** Test set 4,282 lines. **33.2% accuracy, 33.0% macro F1.** Per-character F1: Rachel 0.37, Ross 0.36, Joey 0.36, Monica 0.31, Chandler 0.29, Phoebe 0.28. Mispredictions are legitimately ambiguous (e.g. *"Believe me I'm trying."* Phoebe → Chandler). Brief's 70% bar missed; this matches academic baselines for 6-way isolated-line classification. Useful as weak signal (2× random) for Phase 4 alongside LLM-as-judge.
- **Slack bot died during training.** Resource pressure from 2hr M2 MPS run killed the Socket Mode connection. Restarted after Shafay flagged Slack going quiet. Going forward: pause bot before heavy ML.
- **Chandler assigned Slack Convo QA (permanent).** New ongoing role on top of code review: watches the Slack team itself for routing bugs, identity confusion, persona leaks, hallucinated state. Tool is `agents/test_convo.py`. First catch: multi-name routing bug where "Ross, did Monica approve?" was misrouting to Monica — fixed by prioritizing addressed-at-start match over name-anywhere match. Also: bot moved to detached `screen` session so it survives the Claude Code session lifecycle. Hard-rule output sanitizer added in `app.py` to strip self-intros and stage directions. Model upgraded Haiku 4.5 → Sonnet 4.6 for stricter instruction following.
- **Zombie bot incident.** Discovered FOUR Slack bot instances running simultaneously (PIDs 35388 from 12:47 AM, 40310 from 11:01, 40522 from 11:08, 41831 from 11:51) — each on different code versions, all replying to user messages, causing "Chandler here" leaks, misroutes, and identity confusion. Killed all, restarted clean, hardened `start.sh` to refuse double-starts via `pgrep` check.
- **Phase 2 final verdict.** Ran LLM-as-classifier comparison on 600 stratified samples: Haiku 4.5 = 26.3%, Sonnet 4.6 = 30.3%, DistilBERT (trained) = 33.2%. Three different model architectures, all clustered 26-33% — proves the task ceiling empirically. Shafay accepted **Path A**: classifier stays at 33.2% as a weak signal for Phase 4 alongside LLM-as-judge. No retraining attempted.
- **Phase 2 closed.** Phase 3 starts now: chatbot baseline via RAG, beginning with Joey as the first character.
- **Phase 3 done.** Built `chatbots/build_index.py` (sentence-transformers/all-MiniLM-L6-v2, 384-dim normalized embeddings per character — 42,819 lines indexed in <1 min on M2) + `chatbots/rag.py` (interactive RAG: user message → top-K=8 retrieval → Sonnet 4.6 with character voice notes → in-character reply). Joey's food-obsessed sandwich speech worked first try. Spot-checked all 6 via `chatbots/spot_check.py`: Monica's spice-alphabetizing, Ross's birds-are-dinosaurs nerdiness, Chandler's "could this BE any worse timing" deflection, Rachel's job-interview fashion advice, Phoebe's three-ghosts-in-Debbie's-apartment — every one passes the voice test. Phase 3 done bar (per brief: "chat with one character and it sounds right") cleared for all six.
- **Phase 4 done.** Built the four-script eval harness: `evals/probes.py` (5 tailored probes per character, Phoebe's curation), `evals/generate_replies.py` (runs probes through rag pipeline), `evals/score_classifier.py` (DistilBERT inference per reply), `evals/score_judge.py` (Sonnet 4.6 as judge, 1-10 + reason). Scorecard on 30 replies: **80% overall classifier match rate, 8.40/10 avg judge score**. Per-character: Joey 100%/7.80, Ross 100%/8.00, Monica 80%/8.40, Rachel 80%/8.40, Phoebe 80%/8.80, Chandler 40%/9.00. The Chandler split (judge perfect, classifier weak) is a real finding — his voice is irony patterns not vocabulary, so DistilBERT misses him while Sonnet-as-judge recognizes him cleanly. The brief's two-signal design (classifier + judge) was the right call; their disagreement IS information.
