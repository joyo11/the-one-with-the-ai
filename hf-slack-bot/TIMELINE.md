Mispredictions are legitimately ambiguous (e.g. *"Believe me I'm trying."* Phoebe → Chandler). Brief's 70% bar missed; this matches academic baselines for 6-way isolated-line classification. Useful as weak signal (2× random) for Phase 4 alongside LLM-as-judge.

- **Phase 4 done.** Built the four-script eval harness: `evals/probes.py` (5 tailored probes per character, Phoebe's curation), `evals/generate_replies.py` (runs probes through rag pipeline), `evals/score_classifier.py` (DistilBERT inference per reply), `evals/score_judge.py` (Sonnet 4.6 as judge, 1-10 + reason). ## 2026-06-02

- **Phase 8 kicked off — Pillar 1 (7th Friend identity + memory) shipped.** Supabase project (`pildnxdwywoeubkdmjbt`) created on free tier. Migration 0001: `profiles`, `memories`, `conversations`, `messages` tables with RLS scoped per-user + auto-profile-on-signup trigger. Magic-link auth via `@supabase/ssr` cookies, session middleware refreshes on every request. /auth/callback handles both PKCE (browser-form magic link) and admin-generated token_hash (verifyOtp), so admin bypass links work past the Supabase free-tier email rate limit. Onboarding: 3 questions, all skippable. Auth widget top-right (glass style, aligned with hero nav). Memories review screen at `/memories` — grouped Everyone-knows / per-character, one-tap forget via RLS-scoped DELETE.
- **Pillar 1 wire-up complete: memory reads + writes flow end-to-end.** Chat now proxies through Next.js `/api/chat` (Node runtime). On signed-in chats, server loads `loadProfile` + `loadMemoriesFor(character)`, renders a `## What you remember about <user>` block, sends as `user_context` to the HF backend. `main.py` extended to accept optional `user_context` and append to the character's system prompt. After streaming completes, a fire-and-forget Haiku side channel (`extractAndStoreMemories`) mines durable facts from the user's last message and stores them per-character or with character='*' for shared. Verified live: 5 memories accumulated naturally across one session (e.g. "User graduated from Columbia" / "User owns a Japanese model car"), then Joey referenced "the Japanese car" in a separate session after tab close. Memory persistence across sessions confirmed.
- **Pillar 2 (agents text first) shipped.** Migrations 0002 + 0003: `push_subscriptions` (one per device, endpoint-unique), `scheduled_touches` (queue with sent_at/read_at), profile fields for push_enabled, muted_characters, quiet_hours_start/end, timezone. VAPID keys generated + wired (`NEXT_PUBLIC_VAPID_PUBLIC_KEY` browser, `VAPID_PRIVATE_KEY` server). Service worker at `/sw.js` handles `push` + `notificationclick` events. `/settings` page has notification toggle (Web Push permission + subscribe), per-character mute chips, quiet hours selectors, test-text button. **Two-layer delivery**: (1) in-app inbox — `chat-shell` loads unread `scheduled_touches` on mount, prepends as assistant messages, marks them read; landing picker shows unread-count badges with "You have a text →" CTA; (2) OS push notification as bonus wake-up. `/api/cron/dispatch` walks all push-enabled users, applies 22h throttle + quiet hours (Joey opt-out), picks an unmuted character, generates an in-character "out of the blue" text via Sonnet 4.6 + user memories, inserts the touch, sends push to every device. Scheduled via Supabase `pg_cron` every 2 hours hitting the endpoint with Bearer `CRON_SECRET`. Smoke-tested: endpoint correctly throttles already-touched users + 403s without secret. Generated messages reference memories naturally — e.g. Joey: "Dude are there good sandwich places near Columbia or what 🥪", Phoebe: "ok so my grandma just told me your next job is going to find YOU", Monica: "Just made way too much pasta and I need someone to help me finish it", Ross: dinosaur fact, Rachel: outfit gossip, Chandler: deflecting joke about the job hunt. OS push display has a known macOS-Chrome flakiness (banners don't always surface) but in-app inbox covers that case — messages are never lost.
- **Pillar 3 (living world / calendar) shipped.** Migration 0004: `events` table — global, not user-scoped, RLS read-anyone, character_ids[] array, kind, title, detail, start_at, status. Curated templates per character in `lib/world-seed.ts` (Joey auditions Mon/Wed/Fri, Monica dinner Thu/Sat, Phoebe Friday open mic at Central Perk, Ross museum lectures Tue/Thu, Chandler TPS reports weekdays, Rachel sample sales). Seeder `/api/admin/seed-world` produces 88 events across 28 days (idempotent — skips already-populated days). `lib/world.ts` exposes `pickTouchEvent` (priority: happening-now > soon-upcoming > just-finished) and `renderWeekContext` (injects "## This week in the apartment" into every character's chat system prompt). Touch generation now passes a `worldFraming` block to Claude when an event matches — proactive messages are event-driven instead of vibes-based. Weekly re-seed scheduled via pg_cron (`0 4 * * 0`). Verified live: Monica referenced her seeded "linen closet reorganization" event in a real chat unprompted ("if it was about the linen closet reorganization, I stand by every single decision I made in there").
- **Pillar 4 (translate-your-life / The Lab) shipped.** `/lab` page with 6 character-tools (Joey writes your comeback, Monica reorganizes this, Ross over-explains it, Chandler quips, Rachel rates outfit, Phoebe writes a song).
